MEME_OPERATIONS


 
This PLSQL package implements high level MEME operations.   

assign_cuis 
    FUNCTION assign_cuis ( 
        authority     IN VARCHAR2,
        work_id       IN INTEGER DEFAULT 0,
        table_name    IN VARCHAR2 DEFAULT MEME_CONSTANTS.EMPTY_TABLE,
        new_cui_flag  IN VARCHAR2 DEFAULT MEME_CONSTANTS.NO,
        all_flag      IN VARCHAR2 DEFAULT MEME_CONSTANTS.NO,
        qa_flag       IN VARCHAR2 DEFAULT MEME_CONSTANTS.YES)
    RETURN INTEGER;
This function implements the CUI assignment algorithm.  It can either run on the entire database, or assign for a subset of concepts known to be connected in the MRD sense.  If new_cui_flag is MEME_CONSTANTS.YES, the function will assign official CUIs to concepts which would otherwise have null CUIs (this should be used just prior to a production run).  If new_cui_flag is 'TEMP', the function will assign temporary CUIs to concepts which would otherwise have null CUIs (things like CT0000001).  If all_flag is MEME_CONSTANTS.YES, the function will look at all concepts, and not just those participating in splits and merges.  If qa_flag is MEME_CONSTANTS.YES, the function wil run a few QA checks after assignment to ensure everything went well.  This function is called by $MEME_HOME/bin/assign_cuis.pl.
