UMLS Software Installation Profiles

Overview

The UMLS software development methodology will evolve to the point where all of the software and configuration information used to maintain the data used to produce the Metathesaurus will be tracked by a configruation management system and built into manageable distributable component.  This doucment attempts to describe those distributable components, or "installation profiles".  We have attempted to list all desired profiles, including the environment variables associated with them, the ANT script build targets, the other profiile dependencies, the module lists, and theinstallation instructions.

Here is the current list of installation profiles:
  • Metathesaurus Information Database (MID) Installation
  • Metathesaurus Release Database (MRD) Installation

  • Common Environment Installation
  • External Library Installation
  • Safebox Installation
  • LVG Installation
  • LVG Interface Installation
  • MID Services Installation
  • MID Services Client Installation

  • MEME Application Server Installation
  • MRD Application Server Installation

  • Full MEOW Installation
  • EMS/WMS Installation
  • EMS/WMS Client Installation
  • Jekyll Installation
Later, we will likely also need these:  Recipe Writer Installation.

Metathesaurus Information Database (MID) Installation

Following is information about this installation profile:
Environment Variables:
 $ORACLE_HOME
Description:
 An Oracle instance, fully populated with MID users, schema structures and data.
Dependencies (this machine):
 Oracle Installation
Network Dependencies:
 None
Modules:
 N/A
Ant Build Target: make_initimid (for files to load an initial MID)
Install Instructions:
  1. Create and configure a UTF-8 database.
  2. Consider tune these parameters (for performance):
    • bitmap_merge_area_size
    • create_bitmap_area_size
    • db_block_buffers
    • db_cache_size
    • db_file_multiblock_read_count
    • hash_area_size
    • large_pool_size
    • log_buffer
    • optimizer_dynamic_sampling
    • optimizer_index_caching
    • optimizer_index_cost_adj
    • optimizer_max_permutations
    • parallel_automatic_tuning
    • sga_max_size
    • shared_pool_size
    • sort_area_retained_size
    • sort_area_size
  3. You may be able to increase performance by tuning some OS level parametrs.  for example, this kind of thing in /etc/system may help performance on certain UNIX systems:
    *
    * Allow larger SCSI I/O transfers, parameter is bytes
    *
    set maxphys = 2097152
    *
    * Allow larger Volume Manager I/O transfers, parameter is bytes
    *
    set md_maxphys = 2097152
    *
    * ufs_LW = 1/128th of memory
    * ufs_HW = 1/64th of memory
    *
    set ufs_LW=134217728
    set ufs_HW=268435456

  4. Create MID tablespace (for tables), MIDI tablespace (for indexes)
  5. Create MTH, MEOW schemas (with default MID tablespace)
  6. Grants and roles for MTH:
    GRANT CREATE SEQUENCE TO "MTH";
    GRANT CREATE TABLE TO "MTH";
    GRANT SELECT ANY TABLE TO "MTH" WITH ADMIN OPTION;
    GRANT UNLIMITED TABLESPACE TO "MTH";
    GRANT SELECT ON "SYS"."PUBLICSYN" TO "MTH" WITH GRANT OPTION;
    GRANT "AQ_ADMINISTRATOR_ROLE" TO "MTH";
    GRANT "AQ_USER_ROLE" TO "MTH";
    GRANT "CONNECT" TO "MTH";
    GRANT "DBA" TO "MTH";
    GRANT "DELETE_CATALOG_ROLE" TO "MTH";
    GRANT "EXECUTE_CATALOG_ROLE" TO "MTH";
    GRANT "RESOURCE" TO "MTH";
    GRANT "SELECT_CATALOG_ROLE" TO "MTH";
    GRANT ALTER TABLESPACE TO "MTH";
    GRANT SELECT ANY DICTIONARY TO "MTH"; 
    ALTER USER "MTH" DEFAULT ROLE ALL;
  7. Grants and roles for MEOW:
    GRANT UNLIMITED TABLESPACE TO "MEOW";
    GRANT "CONNECT" TO "MEOW";
    GRANT "DBA" TO "MEOW";
    GRANT "RESOURCE" TO "MEOW";
    ALTER USER "MEOW" DEFAULT ROLE ALL;
  8. Additional Roles (for other users, e.g. editors)
    create role MID_SUPERUSER;
    create role MID_USER;

    -- Initializing MID_SUPERUSER role
    GRANT CONNECT to MID_SUPERUSER with admin option;
    GRANT RESOURCE to MID_SUPERUSER with admin option;
    GRANT DBA to MID_SUPERUSER with admin option;

    -- Initializing MID_USER;
    GRANT CONNECT TO MID_USER;
    GRANT RESOURCE TO MID_USER;
    GRANT EXECUTE ANY PROCEDURE TO MID_USER;
    GRANT "MID_USER" TO "MTH";
    GRANT "MID_USER" TO "MEOW";

  9. Grant ALL on all MTH and MEOW tables to MID_USER
    DECLARE
        CURSOR cur IS SELECT table_name FROM user_tables;
        cv cur%ROWTYPE;
    BEGIN
        OPEN cur;
        LOOP
            FETCH cur INTO cv;
            EXIT WHEN cur%NOTFOUND;
            EXECUTE IMMEDIATE 'GRANT ALL ON ' || cv.table_name || ' TO MID_USER';
        END LOOP;
    END;
    /
  10. Create an $ORACLE_HOME/network/admin/tnsnames.ora name like <machine>_<sid>

  11. If loading a MID from scratch:
    1. Create tables, indexes, views, synonyms, and load packages
      sqlplus user/pwd@db << EOF
      @<initmid>/etc/sql/meme_tables
      @<initmid>/etc/sql/meme_indexes
      @<initmid>/etc/sql/meme_views
      @<initmid>/etc/sql/meme_synonyms
      @<initmid>/etc/sql/meme_packages
      EOF

    2. Populate the tables with initial data (<initmid>/bin/load.csh)
Update Instructions: N/A

MID Services Installation

Following is information about this installation profile:
Environment Variables:
 $MIDSVCS_HOME
Description:
 Key/Value lookup mechanism, supported by a server and a client.  Also includes a tool for obtaining default database authentication information.
Dependencies (this machine):
 $ENV_HOME - used to reference $MIDSVCS_HOME
$SAFEBOX_HOME - used for oracle.passwd file
Network Dependencies:
 None
Modules:
 MID Services file (etc/mid-services-data)
Get Oracle Password Script (bin/get-oracle-pwd.pl)
MID Services Server script (runs as inetd service)
MID Services Client Script (bin/midsvcs.pl)
ROT13 implementation (bin/rot13.pl)
Ant Build Target: make_midsvcs
Install Instructions:
  1. Choose $MIDSVCS_HOME
  2. Unzip dist_midsvcs*.zip into $MIDSVCS_HOME
  3. If you intend to run the server on this machine, configure MID Services File (etc/mid-services-data).
    1. Copy etc/mid-services-data_orig to etc/mid-services-data
    2. Configure etc/mid-services-data
  4. Configure $SAFEBOX_HOME/etc/oracle.passwd file
  5. If you intend to run the server on this machine, first copy bin/midsvcs-server.pl_orig to bin/midsvcs-server.pl.  Then edit:
    1. /etc/services (Add these lines):

      #
      # Local Metathesaurus Services
      #
      midsvcs             5125/tcp        # MID services

    2. (inetd) /etc/inetd.conf (Add these lines):

      #
      # Local Metathesaurus Services
      #
      midsvcs stream tcp nowait nobody /usr/bin/env env \
             ENV_FILE=<PATH TO ENV_HOME>/etc/env.prop \
             ENV_HOME=<PATH TO ENV_HOME> \
             <path to MIDSVCS_HOME>/bin/midsvcs-server.pl

      (xinetd) /etc/xinetd.conf OR /etc/xinetd.d/midsvcs
      #
      # description: key/value lookup service
      #
      service midsvcs
      {
              socket_type     = stream
              user            = meme
              server          = /usr/bin/env
              server_args     = ENV_FILE=/usr/d5/ENV/etc/env.prop ENV_HOME=/usr/d5/ENV /usr/d5/MIDSVCS/bin/midsvcs-server.pl
              wait            = no
              protocol        = tcp
              port            = 5125
      }

    3. Another alternative to the step above would be to hard-code ENV_HOME and ENV_FILE directly in midsvcs-server.pl. Then do this in /etc/inetd.conf instead:

      #
      # Local Metathesaurus Services
      #
      midsvcs stream tcp nowait nobody \
           <path to MIDSVCS_HOME>/bin/midsvcs-server-hardcoded.pl \
            midsvcs-server-hardcoded.pl

    4. Reboot machine or "kill -HUP" on the inetd process (as root)

  6. Edit $ENV_FILE and set MIDSVCS_HOME property to $MIDSVCS_HOME path
  7. Edit $ENV_FILE and set MIDSVCS_HOST and MIDSVCS_PORT properties to the host and port where the MID Services server is running (necessary for installations that do not run a server)
Update Instructions: Same as installation instructions escept for 4a and 4b (which should already have been done).

Ant Build Target: make_meow
Install Instructions:
  1. Choose $MEME_HOME
  2. Unzip dist_meow*.zip into $MEME_HOME
  3. Configure properties filess
    1. Copy the etc/meme.prop_orig to etc/meme.prop
    2. Edit this properties file.
    3. Copy the etc/mrd.prop_orig to etc/mrd.prop
    4. Edit this properties file.
    5. Copy etc/meme.prop to webapps-meme/WEB-INF/classes/meme.prop
    6. Copy etc/mrd.prop to webapps-meme/WEB-INF/classes/mrd.prop
  4. Edit $ENV_FILE and set MEME_HOME property to $MEME_HOME path
  5. Configure Apache
    1. Configure the user info (in $APACHE/conf/httpd.conf). For example, 
      User meme
      Group other
      SetEnv ENV_HOME <path to $ENV_HOME>
      SetEnv ENV_FILE <path to $ENV_FILE>
      SetEnv ORACLE_HOME <path to $ORACLE_HOME>
    2. Configure Timout. For example, 
      Timeout 7200
    3. Configure document root to be $MEME_HOME/www in $APACHE/conf/httpd.conf. For example, 
      DocumentRoot "<path to $MEME_HOME>/www"
    4. Add JNLP to $APACHE/conf/mime.types. For example, 
      application/x-java-jnlp-file    jnlp
    5. Configure CGI directories to point to $MEME_HOME/cgi-bin. 
      #
      # ScriptAlias: This controls which directories contain server scripts.
      # ScriptAliases are essentially the same as Aliases, except that
      # documents in the realname directory are treated as applications and
      # run by the server when requested rather than as documents sent to the client.
      # The same rules about trailing "/" apply to ScriptAlias directives as to
      # Alias.
      #
      ScriptAlias /cgi-bin/ /usr/d5/MEME4/cgi-bin/
      ScriptAlias /cgi-lti-oracle/ /usr/d5/MEME4/cgi-bin/
      ScriptAlias /cgi-oracle-meowusers  /usr/d5/MEME4/cgi-bin/
      ScriptAlias /cgi-oracle-nlmlti  /usr/d5/MEME4/cgi-bin/
      ScriptAlias /cgi-oracle-editors/  /usr/d5/MEME4/cgi-bin/
      AddHandler cgi-script  .cgi
    6. Make sure the CGI <Directory> directive has 
      Options FollowSymLinks
    7. Support server side includes. For example, 
      <IfModule mod_mime.c>
      ...
          #
          # To use server-parsed HTML files
          #
          AddType text/html .shtml
          AddHandler server-parsed .shtml
      ...
      </IfModule>

      <Directory />
          Options FollowSymLinks +Includes
          AllowOverride None
      </Directory>
    8. Support index.shtml as index page. For example, 
      #
      # DirectoryIndex: Name of the file or files to use as a pre-written HTML
      # directory index.  Separate multiple entries with spaces.
      #
      <IfModule mod_dir.c>
          DirectoryIndex index.html index.shtml
      </IfModule>
    9. Configure authentication. For each Directory directive do something like this: 
          Order Deny,Allow
          Deny from all
              AuthType Basic
              AuthName "Metathesaurus Editors Online World"
              AuthUserFile /etc/umls/meow.passwd
              AuthGroupFile /etc/umls/meow.group
              require group meowusers
              deny from redoubt
              satisfy any
    10. Connect Apache, Tomcat.  This is just a suggestion, there are many ways to connect them. See http://jakarta.apache.org/tomcat/tomcat-4.1-doc/config/jk.html for reference. 
      LoadModule jk_module          libexec/mod_jk.so
      #AddModule mod_jk.c

      JkWorkersFile /usr/local/apache/conf/workers.properties
      JkLogFile     /usr/local/apache/logs/mod_jk.log
      JkLogLevel    info
      JkLogStampFormat "[%a %b %d %H:%M:%S %Y] "
      JkMount /*.jsp ajp13
      JkMount /servlet/* ajp13
      JkMount /admin/* ajp13
      JkMount /manager/* ajp13
      JkMount /examples/* ajp13
      JkMount /webapps-meme/* ajp13
      JkMount /webapps-mrd/* ajp13
    11. Next, edit workers.properties. 
      workers.tomcat_home=/usr/local/tomcat
      workers.java_home=/usr/java
      ps=/
      worker.list=ajp13
      worker.ajp13.port=5009
      worker.ajp13.host=localhost
      worker.ajp13.type=ajp13
  6. Configure Tomcat
    1. Connect Apache, Tomcat users.  If you want to use the manager application, you will need a tomcat-users.xml file like this:
      <?xml version='1.0' encoding='utf-8'?>
      <tomcat-users>
        <role rolename="manager"/>
        <user username="mth" password="da3f16e8581e271a13a48dc4468cf1e7" roles="manager"/>
      </tomcat-users>

    2. Connect Apache to Tomcat. First, create a $TOMCAT/webapps/webapps-meme.xml file like this: 
      <Context path="/webapps-meme" docBase="/webapps-meme"
              debug="0">
                <Logger className="org.apache.catalina.logger.FileLogger"
                           prefix="localhost_webapps-meme_log." suffix=".txt"
                        timestamp="true"/>
      </Context>
  7. Configure startup script to start Apache/Tomcat
  8. Configure cron job to run cronvalidate.csh on saturdays. First, copy $MEME_HOME/etc/cronvalidate.csh_orig to $MEME_HOME/bin/cronvalidate.csh. Then configure a crontab entry. For example,
    # Compute MID Validation report each Saturday
    0 9 * * 6 /usr/bin/env ENV_FILE=<path to ENV_HOME>/etc/env.prop \
       ENV_HOME=<path to ENV_HOME> <path to MEME_HOME>/bin/cronvaldate.csh
    An alternative to using "env" is to just call <meme_home>/bin/cronvalidate.csh and edit it to include lines to import the environment like this: 
      setenv ENV_HOME <path to env home>
      setenv ENV_FILE $ENV_HOME/etc/env.prop
      source $ENV_HOME/bin/env.csh
  9. Build tables documentation.  If ant is installed in the environment: cd to $MEME_HOME/etc and run ant using the supplied build.xml file.
