LT TRD - Trade Name Lexical Tag Semantics

 

Overview

As of 2004AB, the only lexical tag (LT) attributes released in the Metathesaurus are TRD, or trade name. There are two important issues with respect to semantics that need  to be explored and refined:
  • 1. At what level do TRD lexical tags exist?  Is it AUI, SUI, or CUI?
  • How should editors approach the task of assigning TRD lexical tags?  Related to this is the issue of how this information appears in concept reports and in the editor

Release Semantics

The current state of lexical tag semantics can be summarized by the following email from Bill Hole (wth@nlm.nih.gov) (Subject: Re: Proposal:  Future of LT attributes, Date: 06/23/2004).

LTs are actually atom properties, and can never be CUI properties! 
  
Think "Kleenex" the brand name vs "Kleenex" the generic name 
(the company blew the process of claiming the rights) to explain 
the Atomic nature of a trademark, or all the drug names which are 
in the same CUI with generic names where only the trademarked 
AUIs are LT=TRD. An LT is NOT a LUI property, either; singular/plural 
and case variation are specific attributes of a trademarked name. 

We currently only send out TRD LTs, though there are more LTs in 
MeSH and the MID.

The only snag came in when editors correctly argued that they 
NEVER want to assign LTs laboriously to every atom, when there 
are many instances of several atoms sharing a SUI which all have 
the same LT. At that point, we decided that IN THE MID 
1- ANY atom assigned an LT triggers propagation of that LT 
to ALL SUI members WITHIN that CUI 
2- Editors need to be able to remove the LT from an atom 
in the rare case like "Kleenex". 
(I can't recall whether this was actually implemented; 
it's another case like Editor Suppressibilty where the 
editor-assigned non-LT status is rare and must always win).
It is also important to mention that RXNORM has its own notion of TRD semantics which imply that a concept is either a "generic" drug (or medical device) or it is a "branded" drug (or device).  In this way, RXNORM sees TRD lexical tags as attributes of a concept, and not of of particular strings.

It is also important to mention that past Metathesaurus semantics caused LT TRD attributes to be assigned at the LUI level.  This was problematic for a variety of reasons and this should never be done again.

Finally TRD LT attributes are released in MRSAT as connected to AUI values with an STYPE of AUI.  However, when an LT attribute exists in the attirbutes table, it is exanded to be connected to every AUI sharing that SUI value.

Editing Semantics

Editors should have a shorthand way of assigning TRD lexical tags so as to not have to attach one to every atom in a concept that is a trade name.  Currently, editors are trained to assume that a TRD lexical tag will apply to the entire concept.  Given that TRD release semantics indicate assignment at the SUI level, this incongruity needs to be resolved.
