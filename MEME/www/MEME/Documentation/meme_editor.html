Adding/Removing a MEME Editor

[ overview | details | references ]

   

Overview/Objective

Periodically new editors need to be added to the system, and periodically old  editors need to be removed.  This document describes the series of steps that should be followed to add or remove and editor from the system.
 

Details

Adding an Editor

When a new editor is to be added to the system, the following steps should be taken:
  • Create new database user.
  • Grant the MID_USER role to this new use.  (MID_USER should be properly configured to allow CONNECT and ALL on all MTH and MEOW tables)
  • Add entry to editors table.  Choose initials to not conflict with another user.
  • Add entry to editor_preferences table.  Use same initials as for editors table.
  • Add an entry with corresponding username/password to the MEOW password file (e.g. /etc/umls/meow.passwd)
  • Add the new user to the "edtiors" group in the /etc/umls/meow.group file).
  • Restart the MEME server.

Deactivating an Editor

This is done by setting the "cur" field in the editors table to N.

UPDATE editors SET cur = 'N' WHERE name = ?;

All queries that access editors should respect this setting when retrieving the list of active editors.

Removing an Editor

Generally, you are discouraged from removing an editor from the system.  You should deactivate the editor instead.  However, if it is important to have the information entirely removed, the following steps can be taken:
  • Remove entry from editors table
  • Remove entry from editor_preferences table
  • MEOW password file (e.g. /etc/umls/meow.passwd)
  • Database user should be removed
  • Restart MEME server
