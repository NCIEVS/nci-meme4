Reporting Server Statistics

Following is documentation on how to have the MEME4 server report some important/interesting statistics.
 
The Statistics As the MEME4 application server runs, it tracks a variety of different statistics that are accessible through this client application.  Following is the current list of statistics reported (note: this list should be updated periodically):
  • Server Start Time.  This is the date that the server was started.
  • Server Run Time. This is the total amount of time that the server has been running for.
  • Free Memory. This is the amount of free memory left on the machine.
  • Memory Used By Server. This is the total amount of memory being used by the server.
  • Total Request Count. This is the total number of properly formatted requests that have entered the system.
  • Active Request Count. This is the count of requests that are actively being processed by the server.
  • Processed Request Count. This is the count of requests that the server has processed to completion (Total - Active should = Processed).
  • ThreadPool Statistics. These are some statistics about the thread pool, including how many threads are being used, how many threads there are total, what the average usage of the threadpool has been, etc.
  • DataSourcePool Statistics. These are some statistics about the data source pool, similar to the thread pool statistics.
  • Requests By Service. This is a histogram of the requests by service name.
  • Requests By Type. This is a histogram of the requests by type (Standard or CGI-style).
  • Exceptions. This is a list of all exceptions that have occurred in the system by type.
Getting the Statistics For your convenience, a client script has been written which connects to the server and requests a formatted report of the statistics listed in the section above.  The default host and port settings for this application come from the midsvcs.pl mappings for meme-server-host and meme-server-port.  Following is the usage, 
% admin.pl -s stats [-host=<host>] [-port=<port>]

 Options:
       -host=<host>:        Name of the machine where server is running 
                              Default is localhost.
       -port=<port>:        Port on which server is listening, default is 8080
       -v[ersion]:          Print version information.
       -[-]help:            On-line help
In it simplest form, all that you have to do is call the script without any arguments.  For example, 
% admin.pl -s stats
If the application server is running on a machine or listening on a port other than the defaults (specified by the midsvcs.pl names meme-server-host and meme-server-port), you will need to invoke admin.pl -s stats with the -host and/or -port switches.  For example, 
% admin.pl -s stats -host oa.nlm.nih.gov -port 8081
If only the host or the port is different, you need only use one of the two switches.
