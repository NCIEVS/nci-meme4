Stamping

Following is documentation on how to run staming.pl.
 
The Environment The client side of the MEME4 system employs a set of potentially confusing properties which are used to configure the part of the client API that interacts directly with the server.   We have taken care to encapsulate all of the necessary environment information into a single properties file which normally resides in $MEME_HOME/bin/meme.prop.  This properties file is a releasable component in MEME4 which means that changes/updates/upgrades to the properties file will be communicated to NLM via the standard release process.

A few key properties can be set via command-line options, including the host and port settings, and an override of the default properties file.
Using the  Script Usage.  
% stamping.pl [-host=<host>] [-port=<port>]
                [-t=<table>] [-f=<file>] [-c=<id list>]
               <db> <authority>

 Options:
       -t=<table>:    Specify a table with a concept_id field 
       -f=<file>:     Specify a file of concept ids.  In the
                      background, this will be loaded into a table
       -c=<id list>:  Specify a comma separated list of concept_ids.
                      Like -f, these ids will be loaded into a table first.
       -host=<host>:  Machine where the application server is running,
                      Default is oa.nlm.nih.gov
       -port=<port>:  Port on which server is listening,
                      Default is 1526
       -v[ersion]:    Print version information.
       -[-]help:      On-line help

In it simplest form, all that you have to do is call the script and pass it a flag indicating where the concept ids to stamp are located, the database (or mid service name) and the authority.  For example, 
% stamping.pl -t t_tostamp editing-db S-BAC

When using this invocation, the table name specified must contain a NUMBER concept_id field.  You may alternatively specify a file name.  For example,

% stamping.pl -f f_tostamp editing-db S-BAC

Here, the file name must contain a single concept id per line.  Finally, you may specify an actual list of concept ids on the command line.  For example,

% stamping.pl -c 2223,2224,2225 editing-db S-BAC

In this case, the actual ids 2223, 2224, and 2225 will be approved. If the application server is running on a machine or listening on the port other than the default (specified by the midsvcs.pl services meme-server-host and meme-server-port) , you will need to invoke insert.pl with the -host and -port switches.  For example, 

% stamping.pl -t t_tosatmp editing-db S-BAC
.
