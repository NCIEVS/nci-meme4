String/Word Normalization

Following is documentation on how to run norm, luiNorm, and wordind.
 
The Environment The NLS environment variable must be set to the MEME_HOME directory for MEME4.  The remaining environment settings are hard coded into the three scripts: norm, luiNorm, and wordind.  LVG is now a Java application, so the CLASSPATH and path to the LVG configuration files must be set correctly within these scripts.  They should be regularly updated as new LVG releases are made.
Using norm Usage.  
% $NLS/bin/norm -h

Synopsis:
  Norm [options]

Description:
  Normalize text.  That is the lvg flow options -f:g:o:t:l:B:w.

Options:
  -d      Print details of Norm operations.
  -h      Print program help information (this is it).
  -hs     Print option's hierarchy structure.
  -i:STR  Define input file name.  The default is screen input.
  -n      Return a "-No Output-" message when an input produces no output.
  -o:STR  Define output file name.  The default is screen output.
  -p      Show the prompt. The default is no prompt.
  -s:STR  Defines a field separator.  The default is "|".
  -t:INT  Define the field to use as the term field.  The default is 1.
  -ti     Display the filtered input term in the output.
  -v      Return the current verion identification of Norm.
  -x:STR  Loading an alternative configuration file.

In it simplest form, all that you have to do is call the script and pass strings to be normalized in on the command line.  The normalized string will be added to the end of the line.  For example, 
% $NLS/bin/norm << EOF
running
playing
EOF
running|run
running|running
playing|play
You may use of the switches shown above to specify things like which field of a delimited file to normalize, and which fields from the input should be written to the output and so forth. 
Using luiNorm
 Usage.  
% $NLS/bin/luiNorm -h

Synopsis:
  LuiNorm [options]

Description:
  Normalize and canonicalize text.  That is the lvg flow options -f:g:o:t:l:B:C:w.

Options:
  -d      Print details of LuiNorm operat
ions.
  -h      Print program help information (this is it).
  -hs     Print option's hierarchy structure.
  -i:STR  Define input file name.  The default is screen input.
  -n      Return a "-No Output-" message when an input produces no outpu
t.
  -o:STR  Define output file name.  The default is screen output.
  -p      Show the prompt. The default is no prompt.
  -s:STR  Defines a field separator.  The default is "|".
  -t:INT  Define the field to use as the term field.  The defau
lt is 1.
  -ti     Display the filtered input term in the output.
  -v      Return the current verion identification of LuiNorm.
  -x:STR  Loading an alternative configuration file.
Usage of luiNorm is identical to usage of norm, it simply guarantees a single normalized form for each input string.  In this way a one-to-one mapping can be created from LUIs to lui-normalized strings.  For example, 
% $NLS/bin/luiNorm << EOF
running
playing
EOF
running|run
playing|play
You may use of the switches shown above to specify things like which field of a delimited file to normalize, and which fields from the input should be written to the output and so forth. 
Using wordind
 Usage.  
% $NLS/bin/wordind -h

Synopsis:
  WordInd [options]

Description:
  Tokenize a term into words.  The "words" are output one per line. That is the lvg flow options -f:l:c.

Options:
  -c      Reserve c
ase (default is to lowercase words).
  -F:INT  Define field to be copied to outputs.
  -h      Print program help information (this is it).
  -hs     Print option's hierarchy structure.
  -i:STR  Define input file name.  The default is screen 
input.
  -n      Return a "-No Output-" message when an input produces no output.
  -o:STR  Define output file name.  The default is screen output.
  -p      Show the prompt. The default is no prompt.
  -s:STR  Defines a field separator.  The 
default is "|".
  -t:INT  Define the field to use as the term field.  The default is 1.
  -v      Return the current verion identification of WordInd.
In its simplest invocation, all you have to do is call the script and pass in strings to tokenize on the command line.  This program basically defines what a "word" is and tokenizes the input stream into its word components.  For example, 
% $NLS/bin/wordind << EOF
Running and playing is what I do.
EOF
running
and
playing
is
what
i
do
You may use of the switches shown above to specify things like which field of a delimited file to tokenize, and which fields from the input should be written to the output and so forth.
