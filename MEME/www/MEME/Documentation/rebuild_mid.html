Rebuilding MID Tables

Following is documentation on how to rebuild the MID tables.
 
The Indexes There are three tables within the MID that track the necessary information required for auto-rebuilding the database tables.  These tables are: (1) meme_tables, which lists all of the tables tracked by the system, (2) meme_indexes, which is a projection of user_indexes for all tables in meme_tables that require indexes, and (3) meme_ind_columns which is a projection of user_ind_columns for all indexes in meme_indexes.

At any time, a snapshot of the existing indexes on MID tables can be taken from user_indexes and user_ind_columns and recorded in meme_indexes and meme_ind_columns by calling the MEME_SYSTEM.refresh_meme_indexes procedure.
Rebuilding For your convenience, a script has been written to rebuild and reindex all tables listed in meme_tables according to the indexes described in meme_indexes and meme_ind_columns. 
% rebuild_mid.pl [-p <num processes>] <database>

 This script is rebuilds all tables in the specified MID.

    Options:
     -p <#>:     Degree of parallelism
     -v[ersion]: Print version information.
     -[-]help:   On-line help
In it simplest form, all that you have to do is call the script and specify a database.  For example, 
% rebuild_mid.pl oa_mid2003
This will start a process that indexes all of the tables with four processes running simultaneously at all times until all tables are rebuilt.  As one process finishes, another is started.  To change the degree of parallelism, use the -p switch. For example, 
% rebuild_mid.pl -p 10 oa_mid2003
Here, instead of having four simultaneous indexing processes, there will be ten processes simultaneously rebuilding the tables until all tables are rebuilt.
