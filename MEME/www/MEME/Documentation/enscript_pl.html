Printing HTML

Following is documentation on how to run enscript.pl.
 
The Environment This application does not require interaction with a MEME4 server.  It does require that the meme.jar file is in the CLASSPATH and that the memerun.pl script can be found in $MEME_HOME/bin.
Using the  Script Usage.  
% jenscript.pl {-1,-2} [-wrap] [-b=<title>] [-p=<printer name>] <file>

 This script is a wrapper around gov.nih.nlm.util.Enscript
 and is used for printing HTML or text documents in MEME4.

 Options:
       -1             Print one page per sheet
       -2             Print two pages per sheet
       -b             Title of the page
       -p             Printer Name
       -wrap          Indicates that document needs to be wrapped
                      with <html>..</html> tags.
       -s             The scaling factor
       -noscale       Indicates that document should not be scaled
       -v[ersion]:    Print version information.
       -[-]help:      On-line help

 Parameters:
       <file>         File nam
In it simplest form, all that you have to do is call the script and pass it the name of the file to print.  For example, 
% jenscript.pl x.html

When using this invocation, default number of pages per sheet is 1, the title is "Untitled" and the user will be presented with a dialog of printers to choose from.  These things can all be explicity specfied.  For example,

% jenscript.pl -2 -b "concept report: 12345" -p lw2 12345.html

Here, two pages will be printed per sheet with the specified title to the printer called "lw2".   The only potentially confusing option is -wrap.  This option exists because code like xreports.pl returns reports which are fragments of HTML documents, intended to be packaged into a larger HTML Document (like the concept report application on meow).  So, if you are sending a concept report to the printer generated by using xreports.pl, it is a good idea to specify -wrap so that the document is wrapped with <html> and <body> tags before printing. For example,

% jenscript.pl -2 -b "concept report: 12345" -p lw2 -wrap 12345.html
A slightly more complicated invocation allows you to scale the document differently (default is to scale with a factor of 1.5 thus shrinking the document by a third).  For certain documents, it may be desirable to either not scale them or to set the scaling factor differently (remember that a higher scaling factor translates to a smaller view on the page).  To print without scaling use this:
 
% jenscript.pl -noscale -b "concept report: 12345" -p lw2 -wrap 12345.html
To set the scaling factor , use an invocation like this:
 
% jenscript.pl -s 2.0 -b "concept report: 12345" -p lw2 -wrap 12345.html
The scaling factor should be expressed as a floating point number.
