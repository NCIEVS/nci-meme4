MEME PL/SQL Packages

 
[ overview | details | references ]
 

 

Overview/Objective

The MEME PL/SQL packages are an implementation of the low-level MEME functions as Oracle PL/SQL procedures.  These functions are logically divided into a series of different packages.

Details

Each package contains a set of functions and procedures that implements a logical portion of the low level MEME services.  Descriptions of each package are shown below as well as the order in which the packages should be loaded (due to dependencies).
 
MEME PL/SQL Package API This document describes a standard set of attributes, procedures, and functions that every MEME_* package should implement.
MEME_CONSTANTS This package contains common constants used by the other packages.  It implements most of the standard constants described here.
MEME_UTILITY This package contains useful utility procedures that make common development tasks simpler.
MEME_SYSTEM This package manages the system (reindexing, table changes, analyzing tables, etc.)
MEME_RANKS This package manages ranking of core table rows and validation of core table field values.
MEME_APROCS This package implements the atomic actions API.
MEME_BATCH_ACTIONS This package implements batch versions of common molecular actions.
MEME_INTEGRITY_PROC This package implements the integrity check procedures and qa_bin procedures.
MEME_SNAPSHOT_PROC This package implements integrity snapshot procedures.
MEME_INTEGRITY This package contains code for running parts of the integrity system like the matrix initializer and the integrity snapshot.
MEME_SOURCE_PROCESSING This package contains code to support the source processing machinery.
MEME_OPERATIONS This package contains code for routine maintenance operations.  Eventually the matrix initializer, integrity snapshot should be run from here.
MEME_TESTER This package contains code to support the testing/validation of the other packages.  For example, it will create sample sets for MEME_BATCH_ACTIONS to work with.

There are dependencies among the packages and so it is important that they be loaded in the correct order.  Currently, that order is:

  • @MEME_CONSTANTS
  • @MEME_UTILITY
  • @MEME_SYSTEM
  • @MEME_RANKS
  • @MEME_APROCS
  • @MEME_BATCH_ACTIONS
  • @MEME_INTEGRITY_PROC
  • @MEME_SNAPSHOT_PROC
  • @MEME_INTEGRITY
  • @MEME_SOURCE_PROCESSING
  • @MEME_OPERATIONS
The $MEME_HOME/sql/packages.sql scrpit can be trusted to load them in the correct order.
