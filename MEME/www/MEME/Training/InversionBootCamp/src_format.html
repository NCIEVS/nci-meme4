Inversion: Source Insertion Input Format

[ overview | details | references ]


Overview/Objective

The SRC files are a collection of specially formatted files used as a standard source insertion input format for the MID.

Details

Following are the SRC Files, including the .raw3 format: 

    • attributes.src
    • classes_atoms.src
    • contexts.src
    • mergefacts.src
    • .raw3 (context format)
    • relationships.src
    • strings.src
    • stringtab.src
    • termgroups.src, sources.src(check with operator to examine current ranks)
The format of the input files is shown below. Fields are separated by the vertical line character ("|").  As of 1/5/2001, the decision was made that each line should be terminated by a "|" character.
 
Source inverters are responsible for the classes_atoms, relationships, .raw3, attributes, mergefacts, termgroups, and sources .src files. The stringtab, strings, and contexts .src files are created by the source insertion software.

Both classes_atoms.src and termgroups.src have suppressible fields so there can be a conflict.  A suppressible_flag of 'Y' in one file overrides a 'N' in the other.  So if a whole termgroup is marked suppressible in termgroups.src, even if those rows in classes are marked 'N', they are considered suppressible.  Similarly, a row marked as suppressible in classes makes that row suppressible even if the termgroup of that row is marked 'N' in the termgroups file.

    source VARCHAR2(20) Range: source_rank 
    integrity_vector VARCHAR2(1500) leave blank for now, but do include as field.
    make_demotion VARCHAR2(1) Range:  {Y,N}
    change_status VARCHAR2(1) Range: {Y,N}
    merge_set VARCHAR2(30) Give it some name; you must have different sets for different combinations of merge_level, integrity_vector, make_demotion and change_status.

    Names should be root SAB name followed by one of the following merge set codes (e.g. MSH-SCUI).
    • SRC -  to be used for mergefacts assembling SRC concepts (no integrities)
    • SY - to be used for source asserted synonymy (generally with integrities, unless we trust source sy more than meta sy)
    • SCUI -special case of SY (based on matching SCUI)
    • AB - special case of SY (based on abbreviations/expansions â€“ no integrities)
    • AE - special case of SY (based on American/british forms â€“ no integrities)
    • EX - special case of SY (based on expansions/short forms â€“ no integrities)
    • LAT - special case of SY (based on translations/ENG atoms â€“ may be integrities)
    • CODE - merging on code (usually with integrities)
    • CUI - merge to UMLS CUI

    This is the starting file format for the context information.  It is similar to MRCXT.  The $MEME_HOME/bin/raw_to_contexts.csh script converts a file like this to the context.src format.  The hcd field of this file has been overloaded to contain additional information, including: source_rui, relationship_group, and the native identifiers (sg_id_1, sg_type_1, sg_qualifier_1, etc.).
     

    .raw3
    source_atom_id_1 NUMBER(12) A valid source atom id (in source_id_map) for this concept.
    context_number NUMBER(12) Atoms can have multiple contexts, this number indicates the n'th context.
    context_level NUMBER(12) Indicates ANC,CCP,CHD, or SIB row (with respect to first field)
    sort_field NUMBER(12) Mechanism for imposing a sort ordering on CHD,SIB rows.
    atom_name VARCHAR2(1200) The string for field 1.
    source_atom_id_2 NUMBER(12) source atom id for this row.
    scd VARCHAR2(30) The code of field1
    hcd VARCHAR2(100) Source asserted hierarchical code.

    Or,

    <hcd>:<source_rui>:<relationship group>~<sg_id_1>~<sg_type_1>~<sg_qual_1>~<sg_id_2>~<sg_type_2>~<sg_qual_2>

    xc NUMBER(1) flag indication child/nochild (1,0)
    source VARCHAR2(20) The source of field 1
    source_of_context VARCHAR2(20)  The source of this context (not necessarily same as source)
    mrrel_flag NUMBER(1) flag indicating whether this row gets into MRREL (1,0).  mostly for SIB rows.
    mrcxt_flag NUMBER(1) flag indicating whether this row gets into MRCXT (1,0).  mostly for SIB rows.
