Questions

  • Why does ICD-9-CM take so long to invert when the update is so small?

    It does not come in a "machine readable" form.  Updates are presented as a PDF document containing descriptions of revisions.  Apelon applies these revisions manually using terminology editing software to an electronic copy of ICD9CM.  The result is then compared to other "standard" ICD9CM data sets and discrepancies are resolved.  This variety of manual steps makes the process very labor intensive, and causes the inversion time to be quite long.

  • Where can you find the "source inversion proposal" for a given source, and what information does it provide?

    Inversion proposals are linked off of the source information pages generated by SIMS.  The source information page index is accessible off of the main page of meow.  All of these files live in (MEOW document root)/Sources

  • Where is the NLM Wiki page for tracking insertion problems?

    The wiki actually lives on unimed but is linked to from the main page on meow.  This URL (MEOW document root)/vqwiki/ will get you there.

  • What are the expanded forms of the acronyms below?
    • UMLS - Unified Medical Language System
    • MID - Metathesaurus Information Database
    • Perl - Practical extraction and reporting language
    • SAX - Simple API for XML
    • XML - eXtensible Markup Language
    • SGML - Standardized Generalized Markup Language
    • MeSH - Medical Subject Headings
    • MEME - Methathesaurus Enhancement and Maintenance Environment
           And, can you use all of them in a single meaningful sentence?
  • Name three important considerations in choosing an SAB (Do you remember what an SAB is?)

    Variety of answers, including:
    1. Must be <= 20 chars
    2. Dependencies based on SAB choice make it difficult to change
    3. Reuse of root SAB value
    4. Consider style of version information

    SAB = Source Abbreviation

  • What 3 forms does an insertion recipe take?

    1. XML
    2. HTML
    3. .csh

  • What insertion steps should not be run during editing?

    Steps that manipulate core data tables or assign identifiers in ranges that overlap with core data fields (either assigning identifiers or mapping the various "SG" identifiers to atom_id/concept_id).

  • Why does inversion require human involvement, even if we have a standard format such as TREF?

    Semantic information and source intent are not encoded in data formats.  There are a variety of questions that still need to be answered in order to produce .src files from TREF.  SRC files also contain some information that TREF is not rich enough to support (such as some source metadata and suppressibility values for the various core data elements).
