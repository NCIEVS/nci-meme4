The Current RxNorm Inversion Proposal

Source inversion proposal for RXNORM_06AA_060120F

I.  General

A.  Outstanding issues

B.  New/changed since last inversion.

Starting with this inversion, all updated sources (RXNORM and NDDF, this time) will be inverted together in one set of *.src files rather than having separate sets of *.src files for each source.

C.  Estimate of new/changed concepts

This release has 362 more RXNORM concepts and 205 more NDDF concepts than the previous versions.

D.  Notes for insertion


II.  Format of input files:

A.  Pointer to source documentation

Documentation: README_Full_01202006.txt

B.  File format:

The file format is the Metathesaurus RRF format.

III.  Inversion plan:

A.  General notes/comments

Three sources were updated this time: RXNORM, NDDF and SNOMEDCT. However, the updated version of SNOMEDCT is already included in the Metathesaurus, so only RXNORM and NDDF were fully inverted as "updated sources".  However, the attributes RXNORM attaches to other sources, including SNOMEDCT, are inverted.

B.  Mapping of source data:

For each field/piece of data in the source files, explain where it will
end up in the inversion.   Note any data that is being ignored.

RXNATOMARCHIVE.RRF: ignored

RXNCONSO.RRF:
    All atoms from updated sources are inverted except for SNOMEDCT, for
    which the current version (2005_07_31) is already in the Metathesaurus
 CUI: inverted as ATN=RXCUI for all sources
 LAT: ignored - currently always ENG
 TS: ignored
 LUI: ignored - currently always blank
 STT: ignored
 SUI: ignored - currently always blank
 ISPREF: ignored
 AUI: inverted as ATN=RXAUI for all sources
 SAUI: used as source_aui (currently blank for non-RXNORM sources)
 SCUI: used as source_cui (currently blank for non-RXNORM sources)
 SDUI: used as source_dui (currently blank for all sources)
 SAB: used as atom's source after mapping to RXNSAB.VSAB
 TTY: used as termgroup
 CODE: used as code
 STR: used (with multiple/leading/trailing blanks removed) as atom_name
 SRL: ignored - currently always blank
 SUPPRESS: used as suppressible flag after mapping blank to "N"
 CVF: ignored - currently always blank

RXNDOC.RRF: EXPL giving expanded_form of new ATNs copied to MRDOC.RRF

RXNREL.RRF:
    Relationships whose SAB is an updated source (other than SNOMEDCT) are
    inverted
 CUI1: used as id_1 where STYPE1="CUI"
 AUI1: used as id_1 where STYPE1="AUI"
 REL: used as relationship_name
 CUI2: used as id_2 where STYPE1="CUI"
 AUI2: used as id_2 where STYPE1="AUI"
 RELA: used as relationship_attribute
 RUI: ignored - currently always blank
 SRUI: used as source_rui (currently always blank)
 SAB: used as both relationship source and source_of_label
 SL: ignored - currently always blank
 RG: ignored - currently always blank
 DIR: ignored - currently always blank
 SUPPRESS: used as suppressible flag after mapping blank to "N"
 CVF: ignored - currently always blank

RXNSAB.RRF:
 VCUI: ignored
 RCUI: ignored
 VSAB: used as VSAB for updated sources
 RSAB: matched to RXNCONSO.SAB to find VSAB
 SON: used as official_name
 SF: used as source_family
 SVER: ignored (instead, version is taken from the part of VSAB after "RXNORM_")
 VSTART: ignored
 VEND: ignored
 IMETA: ignored
 RMETA: ignored
 SLC: used as license_contact
 SCC: used as content_contact
 SRL: used as restriction_level
 TFR: ignored
 CFR: ignored
 CXTY: used as context_type
 TTYL: ignored
 ATNL: ignored
 LAT: used as language
 CENC: used as character_set
 CURVER: ignored
 SABIN: ignored
 SSN: ignored
 SCIT: used as citation

RXNSAT.RRF:
    Attributes whose SAB is an updated source (other than SNOMEDCT) are
    inverted.  Attributes are attached in three ways, depending on the
    source of the atom they're attached to.  For RXNORM atoms, they're
    attached to the RXAUI (ROOT_SOURCE_AUI|RXNORM); for other updated
    sources whose atoms are inverted in classes_atoms.src, they're
    attached to the SOURCE_ATOM_ID; and for other, non-updated sources,
    they're attached to the UMLS AUI.

RXNSTY.RRF:
    All STYs are inverted with source=RXNORM and attached the same three
    ways as the attributes in RXNSAT.

C.  META UI fields:

Indicate what source data (if any) is being used to populate the following fields:

CODE: RXNCONSO.CODE
SAUI: RXNCONSO.RXAUI for RXNORM atoms, RXNCONSO.SAUI for other sources
      (but for other sources, SAUI is currently always blank)
SCUI: RXNCONSO.RXCUI for RXNORM atoms, RXNCONSO.SCUI for other sources
      (but for other sources, SCUI is currently always blank)
SDUI: RXNCONSO.SDUI (currently always blank)

D.  Additional data added during Meta construction

1.  Semantic types.  Document how defaults are assigned

No STYs were added.

2....  Other stuff, e.g. SFO/LFO rels, suppressibility, Americanizations, stripping of UTF-8, etc.

Suppressibility is now taken directly from the SUPPRESS field of RXNCONSO.RRF (after translating blank to "N") rather than being mapped based on TTY, setting RXNORM/OBD and RXNORM/OCD to "O" and other TTYs to "N".
