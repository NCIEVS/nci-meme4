Insertion Recipes

Instructor

Brian Carlsen

Goals

  • What you should learn:
    • The basic forms the recipe takes and the project roles associated with each.
    • The difference between "replacement" and "MID" merging
    • The difference between a "precomputed" merge set and a "generated" merge set
    • The difference between a "source" and "termgroup" safe replacement
  • What you should be able to do:
    • Find the HTML recipe for a source on-line
    • Identify the safe replacement algorithm for a source from the HTML recipe file

Lecture Outline

I. Recipe
   A. XML recipe (in local .src directory)
   B. HTML recipe ((under MEOW document root)/Sources)
   C. Recipe Script (in local .src directory and insertion working directory)
II. Recipe Writing Considerations
   A. New vs. Update Sources
   B. Decisions regarding merging
      i. Matching considerations
      ii. "pre insert" vs "post insert"
   C. Deciding upon safe replacement algorithm
III. Sections and Steps (parameters and defaults)
   A. Load Section
      i. Load Step
   B. Replacement Merge Section (update only)
      i. Precomputed Merge Step
      ii. Generated Merge Step
      iii. Binary Integrity Check Step
      iv. Unary Integrity Check Step
      v. Insert Relationships Based on Matches Step
   C. Safe Replacement Step (update only)
      i. Source Safe Replacement Step
      ii. Termgroup Safe Replacement Step
   D. New Merges Section
      i. MID Merge Step
   E. Post Merge Section
      i. Resolve STYs Step
      ii. Update Releasability Step
      iii. Standard Post Merge Step
   F. Ad Hoc Steps
      i. Generate reports
      ii. Special merge sets
      iii. Create or remove relationships
IV. DEMO
V. Q&A
