Training Documents

Following is a list of available training documents (and descriptions).
 
MEME Training
Introduction and overview of MEME4 based on the March 2005 "MEME Boot Camp"
Inversion Training
 Introduction and overview of UMLS Inversion and Insertion based on the February 2006 "Boot Camp". Click here for a .zip file of all boot camp files. Unzip the .zip file into a local directory and the Powerpoint and other links may work better.
Glossary Definitions of commonly used or confusing terms used in UMLS project.
Table Descriptions List of all active meme tables with detailed descriptions of the fields and usage.
UMLS Data Strucutres A comprehensive review of how UMLS data structures are handled throughout the entire Metathesaurus life-cycle.
Browser Call Notes Notes from browser conference calls.
Source Insertion There are two documents here:
  1. How to insert a source.
  2. How to write a script for a source insertion. (LEGACY DOCUMENTATION)
Research Examples
 This document describes several examples of problems reported by NLM to Apelon and the process used to research and solve the indicated problem.  This is to be a training guide for how to go about answering emails from NLM about potential data problems.
   
Coding Style This document is intended to try and standardize the way code is written. You will see code that does not follow these rules. It is the result of years of programming by different people who did not have a consistent coding style.  If you feel inclined when working with an older piece of code to fix it up to conform to these standards, please do.
There are five subsections:
  1. Perl
  2. csh
  3. SQL
  4. PL/SQL
  5. HTML
  6. Java
Protocols These documents contain descriptions of how to manage some of the common processes, like NLM software releases.  Eventually it should provide links to CGI scripts to facilitate tracking of standard processes like database requests, bug reports, etc.
