Lecture Outline

  1. Actions and Integrities interaction
    1. Editor initiates action
    2. Read concept (important for integrities)
    3. Check integrities (pass, warning or violation)
    4. Perform action
  2. Action Model
    1. Work (work_id)
    2. Activities
    3. Transactions (transaction_id_
    4. Molecular actions (molecule_ids)
    5. Atomic actions (atomic_action_id)
  3. Integrity System
    1. Data integrity checks
    2. Merge inhibitors
    3. Data-driven integrities
      1. single (e.g. MGV_B)
      2. pair (e.g. MGV_K)
    4. Application vectors
    5. Override vectors
  4. Molecular Actions
    1. Insert
    2. Delete
    3. Change
    4. Approve
    5. Merge
    6. Move
    7. Split
  5. Discussion and Questions
