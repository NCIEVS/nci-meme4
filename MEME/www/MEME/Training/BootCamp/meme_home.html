MEME_HOME Code Hierarchy

Code Hierarchy

Following is a brief description of $MEME_HOME and associated important code locations:

Location
 Description
$MEME_HOME This is the base directory for the MEME system.
$MEME_HOME/bin Useful executables for starting and stopping the server as well as managing maintenance and QA processes.  Location of the default properties file.
$MEME_HOME/cgi-bin CGI applications
$MEME_HOME/doc
 javadoc and tables documentation
$MEME_HOME/dump
 Data files for loading an empty new MID
$MEME_HOME/etc
 files for /etc/umls
$MEME_HOME/ext
 Mechanism for "patching" the server.  Jar files placed here are given higher priority in the CLASSPATH when the server is started.
$MEME_HOME/java Contains the class files (in a .jar) and the source files expanded in the subdirectories
$MEME_HOME/log This is typically where server logs are written to.
$MEME_HOME/site Contains files for /site. Should be linke to /site on installed machine
$MEME_HOME/sql  Schema scripts for tables, indexes, and PL/SQL packages
$MEME_HOME/webapps-meme
 JSP web applications
$MEME_HOME/www
 Copy of meow document root (incomplete)
$MEME_HOME/xml
 DTDs
/site/bin Location of perl5, java, and javac.
/site/package/lvg/current Location of current LVG installation
$ORACLE_HOME/network/admin/tnsnames.ora Oracle tnsnames.ora file.
$ORACLE_HOME/rdbms/admin Directory containing Oracle scripts for creating catalog tables like plan_table (utlxplan.sql) or chained_rows (utlchain.sql). (used by schema scripts)
