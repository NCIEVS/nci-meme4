MEME & MRD Java Packages

Package Hierarchy

Following is a brief description of java packages associaged with MEME and MRD projects.

Location
 Description
gov,
gov.nih
gov.nih.nlm
 Top levels, no classes
gov.nih.nlm.util
 Utility classes for things like splitting strings or managing the printing API.  Particularly useful SystemToolkit.
gov.nih.nlm.swing Useful swing components.  Extensions of tables, lists and other components for use in GUI applications.  Particularly useful SwingToolkit.
gov.nih.nlm.meme Useful MEME components.   Toolkit, MID services API, Mail client, etc.  It also tracks version information.
gov.nih.nlm.meme.action Implementation of MEME actions.  Much of this is just for source insertion.
gov.nih.nlm.meme.client MEME Clients.  The "MEME API" is defined here.  This package most completely defines the set of services supported by the MEME server.  Each class here is mirrored by a server-side component.
gov.nih.nlm.meme.common Common objects.  This is the MEME object model for MID data.  The action classes here exist for backwards compatability reasons.
gov.nih.nlm.meme.exception Implementation of MEME exception architecture.
gov.nih.nlm.meme.integrity Implementation of MEME integrity system.
gov.nih.nlm.meme.qa Implement of unit testing.  for development purposes only
gov.nih.nlm.meme.server Server classes themselves, plus classes implementing the available services.  For each client, there is a server side component that performs the actual work. 

Classes for resource pooling and managing sessions are also here.
gov.nih.nlm.meme.sql Interface to the MID.  The data source and connection and action engine classes are the most important.  They read data upon request and cache a variety of metadata in server memory and perform database actions.

The mapper classes define the relational-object mapping (e.g. AttributeMapper).  And the Ticket is a generic, complex parameter passing mechanism.
gov.nih.nlm.meme.xmlClasses for managing XML serialization.
gov.nih.nlm.mrd.client Contains release generator client for building a release.
gov.nih.nlm.mrd.common Mostly objects for managing QA reports and release-level metadata
gov.nih.nlm.mrd.server MRD Server implementation, including handlers for building Metathesaurus release targets.
gov.nih.nlm.mrd.sql Interface to the MRD.
