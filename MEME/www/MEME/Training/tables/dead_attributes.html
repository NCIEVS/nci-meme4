dead_attributes

Overview/Objective

This document contains a description of dead_attributes and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:dead_attributes
Table Type: core
Description:Contains deleted attributes.
Fields:
atom_id NUMBER(12) must be in dead_classes.atom_id 
version_id NUMBER(12) not used 
attribute_id NUMBER(12) unique identifier for each attribute, primary key, a meme_id 
attribute_level VARCHAR2(1) must be in level_status_rank.level_value where table_name = 'A' 
attribute_name VARCHAR2(50) the name of this attribute, e.g. SEMANTIC_TYPE 
attribute_value VARCHAR2(200) the value of this attribute if it can be represented in less than 100 characters. Otherwise it is a pointer into the stringtab.string_id field, e.g. <>Long_Attribute<>:12345 
generated_status VARCHAR2(1) must be in ('Y','N') 
source VARCHAR2(20) must be in source_rank.source 
dead VARCHAR2(1) a Y/N flag indicating whether or not this row is dead 
status VARCHAR2(1) must be in level_status_rank.status where the table_name matches the code for this core table 
authority VARCHAR2(50) the authority responsible for this row, can be validated by MEME_RANKS.get_source_authority_rank 
timestamp DATE indicates when this row was inserted or last modified 
insertion_date DATE indicates the date this row was inserted 
concept_id NUMBER(12) must be in concept_status.concept_id or in dead_concept_status.concept_id 
released VARCHAR2(1) must be in released_rank.released 
tobereleased VARCHAR2(1) must be in tobereleased_rank.tobereleased 
source_rank NUMBER(12) comes from source_rank.rank where the source fields match 
preferred_level VARCHAR2(1) not used 
last_molecule_id NUMBER(12) must be in molecular_actions.molecule_id and is the id of the last molecular action to affect this core table row 
last_atomic_action_id NUMBER(12) must be in atomic_actions.atomic_action_id and is the id of the last atomic action to affect this core table row 
rank NUMBER() the rank of this row, higher values are better ranks 
suppressible VARCHAR2(10) must be in suppressible_rank.suppressible 
atui VARCHAR2(12) must be in attributes_ui.aui 
source_atui VARCHAR2(50) a source asserted unique id for each attribute as it appears in the native source files 
hashcode VARCHAR2(100)  
sg_id VARCHAR2(50) a generic type if identifier, originally for source group identifier 
sg_type VARCHAR2(50) a field indicating the type of an sg_id, valid values can be found in code_map.code where the type='sg_type' 
sg_qualifier VARCHAR2(50) some sg_types require two pieces of information, an id and a qualifier, e.g. for CODE_SOURCE it would be a source value 
sg_meme_data_type VARCHAR2(10) the type of MEME identifier that this element is connected to (e.g. an atom_id, relationship_id, concept_id, attribute_id). The sg_meme_id field will indicate the actual identifier 
sg_meme_id NUMBER(12) the actual MEME identifier that this element is connected to (e.g. an atom_id, relationship_id, concept_id, attribute_id). The sg_meme_data_type field will indicate what type of identifier it is 
Indexes:
  • dead_attributes_pk ON attribute_id
    • Editing Role:When MEME_APROCS.aproc_change_dead is called to delete an attribute, a row gets inserted into this table
