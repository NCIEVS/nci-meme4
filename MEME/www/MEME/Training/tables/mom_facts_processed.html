mom_facts_processed

Overview/Objective

This document contains a description of mom_facts_processed and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:mom_facts_processed
Table Type: source
Description:An archive of all of the merge facts that have been processed in a given calendar year. All facts are stored here, including ones that fail to merge, produce demotions, or are already merged.
Fields:
merge_fact_id NUMBER(12) a unique id within a merge set 
atom_id_1 NUMBER(12) must be in classes.atom_id or in dead_classes.atom_id 
merge_level VARCHAR2(3) a value indicating the type of merge fact, usually one of MAT (exact match), NRM (norm match), or SY (synonym) 
atom_id_2 NUMBER(12) must be in classes.atom_id or in dead_classes.atom_id 
source VARCHAR2(20) the source of this fact 
integrity_vector VARCHAR2(1500) a vector of integrity checks, e.g. <MGV_C:V><MGV_H1:V> 
make_demotion VARCHAR2(1) a Y/N flag indicating whether or not demotion relationships should be created if this merge fact fails 
change_status VARCHAR2(1) a Y/N flag indicating whether or not the atoms merged by this fact should be unapproved 
authority VARCHAR2(50) the authority responsible for this row, can be validated by MEME_RANKS.get_source_authority_rank 
merge_set VARCHAR2(30) the name of the merge set that this fact belongs to 
violations_vector VARCHAR2(1500) an integrity vector showing which checks failed to pass 
status VARCHAR2(1) see mom_merge_facts 
merge_order NUMBER(12) a key used to order merge facts 
molecule_id NUMBER(12) the molecule_id responsible for the merging of this fact 
work_id NUMBER(12) must be either 0 or must be in meme_work.work_id 
Indexes:
  • x_mfp_s_ms ON source, merge_set
    • Insertion Role:After a merge set has been processed, the resulting facts are loaded into this table by MEME_SOURCE_PROCESSING.move_processed_facts.
