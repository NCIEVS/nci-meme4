relationships_ui

Overview/Objective

This document contains a description of relationships_ui and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:relationships_ui
Table Type: ui
Description:Maps sets of fields to unique relationship identifiers. This table is used like string_ui in the sense that it tracks RUI assignments historically over time. The RUI is intended as a versionless relationship identifier.
Fields:
source_rui VARCHAR2(50) a source asserted unique id for each relationship as it appears in the native source files (e.g. SNOMED RELATIONSHIPID) 
rui VARCHAR2(10) unique identifier for each versionless relationship, also a primary key. A versionless relationship is represented by the other fields of this table. 
root_source VARCHAR2(20) a value representing a source independent of version which can be used across time to mean the same source 
relationship_level VARCHAR2(1) must be in level_status_rank.level_value where table_name = 'R' 
relationship_name VARCHAR2(10) must be in inverse_relationships.relationship_name 
relationship_attribute VARCHAR2(100) must be in inverse_rel_attributes.relationship_attribute 
sg_id_1 VARCHAR2(50) a generic type if identifier, originally for source group identifier 
sg_type_1 VARCHAR2(50) a field indicating the type of an sg_id, valid values can be found in code_map.code where the type='sg_type' 
sg_qualifier_1 VARCHAR2(50) some sg_types require two pieces of information, an id and a qualifier, e.g. for CODE_SOURCE it would be a source value 
sg_id_2 VARCHAR2(50) a generic type if identifier, originally for source group identifier 
sg_type_2 VARCHAR2(50) a field indicating the type of an sg_id, valid values can be found in code_map.code where the type='sg_type' 
sg_qualifier_2 VARCHAR2(50) some sg_types require two pieces of information, an id and a qualifier, e.g. for CODE_SOURCE it would be a source value 
Indexes:
  • x_rui_rui ON rui
  • x_rui_sg_id_1 ON sg_id_1, sg_type_1
  • x_rui_sg_id_2 ON sg_id_2, sg_type_2
    • Insertion Role:When relationships is loaded by the source insertion process, any combination of stripped source, level, name, attribute, and connected ids that have never yet existed in the system are assigned RUI values and loaded into this table. The RUi for the inverse is also loaded. This operation is performed by MEME_SOURCE_PROCESSING.assign_ruis.
