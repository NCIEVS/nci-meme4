meme_work

Overview/Objective

This document contains a description of meme_work and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:meme_work
Table Type: system
Description:A record of large database operations. Insertions, Matrix init runs, CUI assignments and other maintenance work should request a work_id and log in this table the details of the work at hand. This is done by calling MEME_UTILITY.new_work. Eventually editing should be tracked through this table as well so that anywhere that one encounters a work_id, it could be looked up here to find a description of what work was involved.
Fields:
work_id NUMBER(12) a unique id representing some amount and some kind of database work 
timestamp DATE indicates when this row was inserted or last modified 
type VARCHAR2(20) a general indicator of the type of work done, usually one of: INITIALIZE, MATRIX, MAINTENANCE, INSERTION (each of these has a MEME_CONSTANTS variable) 
authority VARCHAR2(50) the authority of the person or process who is performing the work 
description VARCHAR2(1000) a detailed description 
Indexes: No indexes
Insertion Role:Every source insertion has a work_id associated with it that can be found here. 
Editing Role:Every matrix initializer and cui assignment run has a work_id associated with it. Eventually, editor sessions, or editors should be assigned work_ids so that their work over time can be tracked in the action tables.
