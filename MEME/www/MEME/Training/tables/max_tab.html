max_tab

Overview/Objective

This document contains a description of max_tab and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:max_tab
Table Type: auxiliary
Description:Maintains the maximum id values for various kinds of IDs. Each row has a table_name which specifies what kind of id is being tracked and a maximum value. The value in the max_id field is the maximum id used so far, so if is 12345 for table "ATOMS" it means that the maximum atom_id used so far is 12345.
Fields:
table_name VARCHAR2(50) a label used to track a max_id, e.g. ATOMIC_ACTIONS tracks the highest atomic_actions.atomic_action_id 
max_id NUMBER(12) the maximum id, to get the next id add 1 to this 
Indexes: No indexes
Insertion Role:When new core data is added during the insertion process, the meme ids used are generated from the max values held in this table. The same goes for the new string identifiers (SUI, LUI, ISUI). 
Editing Role:Every time an action happens, the action identifiers are incremented in this table and used to log the action. Each time a piece of core data is inserted, or a new string is inserted, the max identifiers for those things are incremented and used. 
Production Role:When new CUIs are assigned for production, this table tracks the previous max CUI value and all new ones are assigned in the range immediately beyond that one.
