meme_schedule

Overview/Objective

This document contains a description of meme_schedule and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:meme_schedule
Table Type: system
Description:An advanced table used for scheduling server processes. This table is complex to use. In a generic sense, it is used to inform server applications of various different run and lock mode states at different times. Another "listener" type of application must be used that understands the semantics of this table.
Fields:
application VARCHAR2(100) the name of an application or component in the MEME system 
specific_date DATE a value in this field is used to have a schedule entry apply to a specific date, e.g. 01-jan-2002 
day_of_week NUMBER(12) a 0-6 (sun-sat) value indicates that the schedule entry should apply to that day of the week 
start_time VARCHAR2(10) a value in here indicates that the schedule entry applies beginning at this time of day 
end_time VARCHAR2(10) a value in here indicates that the schedule entry ceases to apply at this time of day 
cpu_mode VARCHAR2(20) the cpu mode that an application should run under if the conditions are satisfied, one of: SHUTDOWN, STANDBY, DELAY, NODELAY 
lock_mode VARCHAR2(20) not used yet, but will have values that indicate to applications whether or not it is OK to lock core data tables 
Indexes: No indexes
