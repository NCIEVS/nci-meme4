extraction_history

Overview/Objective

This document contains a description of extraction_history and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:extraction_history
Table Type: unknown
Description:This table is used by the MRDSyncManager to remember what data changes have been extracted from the MID. When the application starts, it looks up the data in this table to determine where to start extracting new data changes.
Fields:
work_id NUMBER(12) must be either 0 or must be in meme_work.work_id 
authority VARCHAR2(20) the authority responsible for this row, can be validated by MEME_RANKS.get_source_authority_rank 
timestamp DATE indicates when this row was inserted or last modified 
first_mid_event_id NUMBER(12)  
last_mid_event_id NUMBER(12)  
row_count NUMBER(12)  
valid_extraction VARCHAR2(1)  
Indexes:
  • x_eh_lmidid ON last_mid_event_id
