connected_concepts

Overview/Objective

This document contains a description of connected_concepts and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:connected_concepts
Table Type: unknown
Description:Tracks connections between concepts arising from atomic_actions which change aui, aui_1, aui_2, concept_id, concept_id_1 or concept_id_2. See MEME_APROCS.connect_concepts.
Fields:
concept_id_1 NUMBER(12) must be in concept_status.concept_id or in dead_concept_status.concept_id 
concept_id_2 NUMBER(12) must be in concept_status.concept_id or in dead_concept_status.concept_id 
Indexes:
  • x_con_cn1 ON concept_id_1, concept_id_2
  • x_con_cn2 ON concept_id_2, concept_id_1
