atoms_ui

Overview/Objective

This document contains a description of atoms_ui and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:atoms_ui
Table Type: ui
Description:Maps sets of fields to unique identifiers. This table is used like string_ui in the sense that it tracks AUI assignments historically over time. The AUI is intended as a versionless atom identifier (somewhere between a SUI and an atom id).
Fields:
source_aui VARCHAR2(50) a source asserted unique id for each atom as it appears in the native source files (e.g. SNOMED DESCRIPTIONID) 
source_cui VARCHAR2(50) a source asserted unique id for each concept as it appears in the native source files (e.g. SNOMED CONCEPTID) 
source_dui VARCHAR2(50) a source asserted unique id for each descriptor as it appears in the native source files (e.g. MeSH D#) 
aui VARCHAR2(10) unique identifier for each versionless atom, also a primary key. A versionless atom is represented by the other fields of this table. 
sui VARCHAR2(10) must be in string_ui.sui 
stripped_source VARCHAR2(20) a value representing a source independent of version which can be used across time to mean the same source 
tty VARCHAR2(20) the term type, must be in termgroup_rank.tty 
code VARCHAR2(50) a value assigned to this atom/term by the source provider, this value has semantics specific to that source provider 
Indexes:
  • atoms_ui_pk ON aui
  • x_aui_sui ON sui
    • Insertion Role:When classes is loaded by the source insertion process, any combination of stripped source, sui, term type and code that have never yet existed in the system are assigned AUI values and loaded into this table. This operation is performed by MEME_SOURCE_PROCESSING.assign_auis.
