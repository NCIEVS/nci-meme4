source_concept_status

Overview/Objective

This document contains a description of source_concept_status and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:source_concept_status
Table Type: source
Description:Staging area for new concepts.
Fields:
source VARCHAR2(20) must be in source_rank.source 
status VARCHAR2(1) must be in level_status_rank.status where the table_name matches the code for this core table 
released VARCHAR2(1) must be in released_rank.released 
tobereleased VARCHAR2(1) must be in tobereleased_rank.tobereleased 
concept_id NUMBER(12) must be in concept_status.concept_id or in dead_concept_status.concept_id 
preferred_atom_id NUMBER(12) the id of the preferred atom for this concept, must be in classes.atom_id or in dead_classes.atom_id 
switch VARCHAR2(1) a flag indicating whether or not this row is to be inserted, R means yes 
source_concept_id NUMBER(12) not used 
cui VARCHAR2(10) a Concept Unique Identifier, e.g. C0000039 
Indexes:
  • x_scs_1 ON concept_id
    • Insertion Role: Data is generated from the source_classes_atoms table, then the concepts are loaded into concept_status.
