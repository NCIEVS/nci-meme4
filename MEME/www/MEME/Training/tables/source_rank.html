source_rank

Overview/Objective

This document contains a description of source_rank and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:source_rank
Table Type: auxiliary,metadata
Description:Maintains metadata information about the sources that contribute to Metathesaurus content. This table also serves as a mechanism to normalize valid source values.
Fields:
source VARCHAR2(20) Unique id, primary key 
rank NUMBER(12) the rank of this row, higher values are better ranks 
restriction_level NUMBER(12) a 0-3 value indicating what type of licensing arrangement is necessary to use this source 
normalized_source VARCHAR2(20) used to map a source value that does not have an SRC concept to one that does, e.g. MSH2002HMCE has a normalized source of MSH2002 
stripped_source VARCHAR2(20) a value representing a source independent of version which can be used across time to mean the same source 
source_family VARCHAR2(20) a value representing a group of sources that are connected to one another, e.g. CPT and MTHCH have the same source family: CPT 
version VARCHAR2(20) the version number of a component/application 
notes VARCHAR2(1000) any additional information 
Indexes:
  • source_rank_pk ON source
    • Insertion Role:When data for a new or update source is going to be inserted, we first insert information about that source into this table. The data is loaded from source_source_rank. 
    Editing Role:During the editing cycle, this table must be maintained using the SRC Info Editor. Much of the semantics of this table can be validated using the MID Validation System. 
    Production Role:This is the source of data for MRSAB.
