inverse_rel_attributes

Overview/Objective

This document contains a description of inverse_rel_attributes and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:inverse_rel_attributes
Table Type: auxiliary,metadata
Description:Maps relationship attributes to their inverses. Additionally it allowed for the ranking of relationship attributes so that it can participate in the overall rank of the relationship.
Fields:
relationship_attribute VARCHAR2(100) a valid relationship attribute, primary key 
inverse_rel_attribute VARCHAR2(100) the inverse rela, e.g. the inverse of mapped_from is mapped_to 
rank NUMBER(12) the rank of this row, higher values are better ranks 
Indexes: No indexes
Editing Role:When reading a concept from the database for a given concept_id, relationships must be read in both directions (where concept_id_1=concept_id and where concept_is_2=concept_id). When reading the inverse direction (where the current concept we want is the concept_id_2 of the relationship), we must reverse the sense of the relationship attribute (e.g. mapped_to becomes mapped_from) by mapping it through this table.
