relationships

Overview/Objective

This document contains a description of relationships and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:relationships
Table Type: core
Description:Contains relationships between atoms and concepts. This is a core table. This table is very intimately connected with classes and sg_relationships.
Fields:
relationship_id NUMBER(12) unique identifier for each relationship, primary key, a meme_id 
version_id NUMBER(12) not used 
relationship_level VARCHAR2(1) must be in level_status_rank.level_value where table_name = 'R' 
atom_id_1 NUMBER(12) must be in classes.atom_id for non concept level relationships 
relationship_name VARCHAR2(10) must be in inverse_relationships.relationship_name 
relationship_attribute VARCHAR2(100) must be in inverse_rel_attributes.relationship_attribute 
atom_id_2 NUMBER(12) must be in classes.atom_id for non concept level relationships 
source VARCHAR2(20) must be in source_rank.source 
generated_status VARCHAR2(1) must be in ('Y','N') 
dead VARCHAR2(1) a Y/N flag indicating whether or not this row is dead 
status VARCHAR2(1) must be in level_status_rank.status where the table_name matches the code for this core table 
authority VARCHAR2(50) the authority responsible for this row, can be validated by MEME_RANKS.get_source_authority_rank 
timestamp DATE indicates when this row was inserted or last modified 
insertion_date DATE indicates the date this row was inserted 
concept_id_1 NUMBER(12) must be in classes.concept_id 
concept_id_2 NUMBER(12) must be in classes.concept_id 
released VARCHAR2(1) must be in released_rank.released 
tobereleased VARCHAR2(1) must be in tobereleased_rank.tobereleased 
source_rank NUMBER(12) comes from source_rank.rank where the source fields match 
preferred_level VARCHAR2(1) not used 
last_molecule_id NUMBER(12) must be in molecular_actions.molecule_id and is the id of the last molecular action to affect this core table row 
last_atomic_action_id NUMBER(12) must be in atomic_actions.atomic_action_id and is the id of the last atomic action to affect this core table row 
rank NUMBER() the rank of this row, higher values are better ranks 
source_of_label VARCHAR2(20) must be in source_rank.source 
suppressible VARCHAR2(10) must be in suppressible_rank.suppressible 
rui VARCHAR2(12) must be in relationships_ui.rui 
source_rui VARCHAR2(50) a source asserted unique id for each relationship as it appears in the native source files (e.g. SNOMED RELATIONSHIPID) 
relationship_group VARCHAR2(10) a set or group_id that indicates that multiple relationships with the same concept_id_1/cui_1 belong together 
sg_id_1 VARCHAR2(50) a generic type if identifier, originally for source group identifier 
sg_type_1 VARCHAR2(50) a field indicating the type of an sg_id, valid values can be found in code_map.code where the type='sg_type' 
sg_qualifier_1 VARCHAR2(50) some sg_types require two pieces of information, an id and a qualifier, e.g. for CODE_SOURCE it would be a source value 
sg_meme_data_type_1 VARCHAR2(10) the type of MEME identifier that this element is connected to (e.g. an atom_id, relationship_id, concept_id, attribute_id). The sg_meme_id field will indicate the actual identifier 
sg_meme_id_1 NUMBER(12) the actual MEME identifier that this element is connected to (e.g. an atom_id, relationship_id, concept_id, attribute_id). The sg_meme_data_type field will indicate what type of identifier it is 
sg_id_2 VARCHAR2(50) a generic type if identifier, originally for source group identifier 
sg_type_2 VARCHAR2(50) a field indicating the type of an sg_id, valid values can be found in code_map.code where the type='sg_type' 
sg_qualifier_2 VARCHAR2(50) some sg_types require two pieces of information, an id and a qualifier, e.g. for CODE_SOURCE it would be a source value 
sg_meme_data_type_2 VARCHAR2(10) the type of MEME identifier that this element is connected to (e.g. an atom_id, relationship_id, concept_id, attribute_id). The sg_meme_id field will indicate the actual identifier 
sg_meme_id_2 NUMBER(12) the actual MEME identifier that this element is connected to (e.g. an atom_id, relationship_id, concept_id, attribute_id). The sg_meme_data_type field will indicate what type of identifier it is 
Indexes:
  • relationships_pk ON relationship_id
  • x_r_a1 ON atom_id_1
  • x_r_a2 ON atom_id_2
  • x_r_c1 ON concept_id_1
  • x_r_c2 ON concept_id_2
    • Insertion Role:Relationships are one type of core data loaded during the source insertion process. There are several steps:
    • Load the data into the source_context_relationships table from a relationships.src file.
    • Assign meme ids
    • Insert source_id_map rows
    • Map the source data to atom and concept ids
    • Insert the rows into the live relationships table
     
    Editing Role:This is one kind of core data that editors can manipulate with molecular actions. 
    Production Role:This table is the source of data for MRREL and for the bequeathal rels in MRCUI.
