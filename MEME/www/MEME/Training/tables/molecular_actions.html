molecular_actions

Overview/Objective

This document contains a description of molecular_actions and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:molecular_actions
Table Type: action
Description:The log of editor actions. Each row in this table is a log of a higher level action on the core data in the MID.
Fields:
transaction_id NUMBER(12) this is an id that groups molecular actions together so they can be retrieved as a set 
molecule_id NUMBER(12) a unique identifier for each row, primary key 
authority VARCHAR2(50) the authority of the person or process who performed the action 
timestamp DATE indicates when this row was inserted or last modified 
molecular_action VARCHAR2(30) a code indicating the action performed, must be in code_map where the type='molecular_action' 
source_id NUMBER(12) the id of the primary concept involved in an action 
target_id NUMBER(12) the id of the secondary concept involved in an action, e.g. the target of a merge operation 
undone VARCHAR2(1) a Y/N flag indicating whether or not this action was undone 
undone_by VARCHAR2(20) the authority responsible for undoing this action, if undone='Y' 
undone_when DATE the timestamp of the undoing of this action, if undone='Y' 
status VARCHAR2(1) an R/N flag indicating whether or not this action is approved 
elapsed_time NUMBER(12) an integer number of milliseconds 
work_id NUMBER(12) must be either 0 or must be in meme_work.work_id 
Indexes:
  • molecular_actions_pk ON molecule_id
  • x_ma_sid ON source_id
  • x_ma_tid ON target_id
  • x_ma_trid ON transaction_id
    • Insertion Role:Any actions performed during source insertion are logged as either MOLECULAR or MACRO actions. 
    Editing Role:During the editing cycle, actions performed by automated processes or by editors are logged here using one row per action.
