mrd_contexts

Overview/Objective

This document contains a description of mrd_contexts and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:mrd_contexts
Table Type: unknown
Description:This table tracks mrd states for contexts. The data is derived from the PAR rows in the context_relationships table. Note, since the data comes from PAR rows, it is not obvious that the context information for treetops will be in this table, but they are.
Fields:
aui VARCHAR2(10) must be in atoms_ui.aui 
parent_treenum VARCHAR2(1000) a tree-number composed of auis separated by periods (.), indicates the tree-position of the atom_id_2 in aui terms 
root_source VARCHAR2(20) a value representing a source independent of version which can be used across time to mean the same source 
hierarchical_code VARCHAR2(1000) the tree-number given to the atom_id_1 by the source provider 
relationship_attribute VARCHAR2(100) must be in inverse_rel_attributes.relationship_attribute 
release_mode VARCHAR2(10) a flag indicating whether or not siblings of this row should be released in MRREL and/or MRCXT 
rui VARCHAR2(12) must be in relationships_ui.rui 
source_rui VARCHAR2(50) a source asserted unique id for each relationship as it appears in the native source files (e.g. SNOMED RELATIONSHIPID) 
relationship_group VARCHAR2(10) a set or group_id that indicates that multiple relationships with the same concept_id_1/cui_1 belong together 
insertion_date DATE indicates the date this row was inserted 
expiration_date DATE  
Indexes:
  • x_mcxt_aid ON aui
  • x_mcxt_ptn ON parent_treenum
