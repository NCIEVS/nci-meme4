source_string_ui

Overview/Objective

This document contains a description of source_string_ui and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:source_string_ui
Table Type: source
Description:Staging area for new names.
Fields:
atom_id NUMBER(12) must be in classes.atom_id or in dead_classes.atom_id 
lui VARCHAR2(10) must be in string_ui.lui 
sui VARCHAR2(10) must be in string_ui.sui 
string_pre VARCHAR2(30) the first ten characters of the string, used for indexing 
norm_string_pre VARCHAR2(30) the first ten characters of the norm_string, used for indexing 
language VARCHAR2(20) the name of a language 
base_string VARCHAR2(1) a Y/N flag indicating whether or not this is a base string 
string VARCHAR2(3000) a name that has appeared in the Metathesaurus 
norm_string VARCHAR2(3000) the result of running string through the lvg luiNorm program 
isui VARCHAR2(10) must be in string_ui.isui 
lowercase_string_pre VARCHAR2(30) the first ten characters of the string (lowercased), used for indexing 
Indexes:
  • x_ssu_lp ON lowercase_string_pre
  • x_ssu_nsp ON norm_string_pre
  • x_ssu_sp ON string_pre
  • x_ssui_string ON string
    • Insertion Role: First, they are loaded from strings.src, then the strings are prepared for insertion and finally they are loaded into string_ui.
