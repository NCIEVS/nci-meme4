sr_predicate

Overview/Objective

This document contains a description of sr_predicate and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:sr_predicate
Table Type: auxiliary
Description:Contains descriptions of the matching predicates used to compute safe replacement for the various current sources in the MID.
Fields:
source VARCHAR2(20) must be in source_rank.source 
row_sequence NUMBER(12) a positive integer used for ordering rows of the table 
current_name VARCHAR2(20) the name of the current version of a source in the Metathesaurus 
previous_name VARCHAR2(20) the name of the immediately previous version of a source that has been updated 
string_match VARCHAR2(10) a value indicating how strings were matched in the safe replacement predicate 
code_match VARCHAR2(10) a value indicating how codes were matched in the safe replacement predicate 
tty_match VARCHAR2(10) a value indicating how ttys were matched in the safe replacement predicate 
source_cui_match VARCHAR2(10)  
replaced_termgroups VARCHAR2(2000) a comma separated list of obsolete termgroups that were replaced 
replacement_termgroups VARCHAR2(2000) a comma separated list of new termgroups that were replace obsolete ones 
replaced_sources VARCHAR2(2000) a comma separated list of obsolete sources that were replaced 
replacement_sources VARCHAR2(2000) a comma separated list of new sources that were replace obsolete ones 
Indexes: No indexes
Insertion Role:During the insertion of an update source, this table is populated by one or more calls to MEME_SOURCE_PROCESSING.safe_replacement.
