stringtab

Overview/Objective

This document contains a description of stringtab and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:stringtab
Table Type: auxiliary
Description:Contains long attribute values. The attributes table has only 100 characters alloted for attribute_value. For any attributes with longer attribute values (such as definitions), a pointer into the stringtab table is constructed of the form '<>Long_Attribute<>:<string_id>' where <string_id> is a valid string id in this table.
Fields:
string_id NUMBER(12) an id that groups fragments of a long string together 
row_sequence NUMBER(12) a positive integer used for ordering rows of the table 
text_total NUMBER(12) the length of a string when all of the fragments are put together 
text_value VARCHAR2(3000) one fragment of a potentially larger string, bounded to 1786 by OpenROAD 
Indexes:
  • stringtab_pk ON string_id, row_sequence
    • Insertion Role:When attributes are loaded by the source insertion process, any rows that have an attribute value longer than 100 characters also get loaded here. 
    Production Role:Data from this table is used in the production of MRDEF and MRSAT that have values greater than 100 characters.
