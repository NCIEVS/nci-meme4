ic_override

Overview/Objective

This document contains a description of ic_override and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:ic_override
Table Type: integrity
Description:Maps editor levels to override integrity vectors. An application like the editing interface will start with a default integrity vector and then for each editor (or editor level) allows an override vector which specifies integrity check levels for certain checks that should override the default. For example, if the default vector says <MGV_H1:E> then by default check MGV_H1 is enforced, however you may want certain higher level editors to not be subjected to this constraint in which case you can specify an override vector having this <MGV_H1:> which will produce only a warning instead of a violation
Fields:
ic_level NUMBER(12) an editor level value used to determine the override vector for different editors 
override_vector VARCHAR2(2000) a type of integrity vector used to override integrity check settings 
Indexes: No indexes
Editing Role:By default, the editing interface is configured to override the default integrity vector with the override vector from this table matching the editor's editor level
