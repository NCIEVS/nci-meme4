meme_cluster_history

Overview/Objective

This document contains a description of meme_cluster_history and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:meme_cluster_history
Table Type: integrity
Description:Maintains map of "connections" between concepts based on integrity violations. This table is used by the matrix updater to track relationships between concepts that need to be explored when computing the subset of the database that it needs to operate on.
Fields:
name VARCHAR2(20) the name of a qa check 
concept_id_1 NUMBER(12) must be in concept_status.concept_id or in dead_concept_status.concept_id 
concept_id_2 NUMBER(12) must be in concept_status.concept_id or in dead_concept_status.concept_id 
Indexes:
  • x_mch_1 ON concept_id_1
  • x_mch_2 ON concept_id_2
