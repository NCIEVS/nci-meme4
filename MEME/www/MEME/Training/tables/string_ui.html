string_ui

Overview/Objective

This document contains a description of string_ui and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:string_ui
Table Type: ui
Description:Contains one row for each string that has ever been in the MID. This table is only ever inserted into (unless QA problems require updates/deletes). It maintains a record of every string that has ever been present in the MID or in the Metathesaurus and maps those strings to a normalized string, a SUI, a LUI, and an ISUI.
Fields:
lui VARCHAR2(10) a lexical class identifier, it is a code that represents the norm_string 
sui VARCHAR2(10) a string unique identifier, it is a code that represents the string 
string_pre VARCHAR2(30) the first ten characters of the string, used for indexing 
norm_string_pre VARCHAR2(30) the first ten characters of the norm_string, used for indexing 
language VARCHAR2(20) the name of a language 
base_string VARCHAR2(1) a Y/N flag indicating whether or not this is a base string 
string VARCHAR2(3000) a name that has appeared in the Metathesaurus 
norm_string VARCHAR2(3000) the result of running string through the lvg luiNorm program 
isui VARCHAR2(10) a case insensitive string unique identifier, it is a code that represents the lowercased string 
lowercase_string_pre VARCHAR2(30) the first ten characters of the string (lowercased), used for indexing 
Indexes:
  • string_ui_pk ON sui
  • x_su_isui ON isui
  • x_su_lp ON lowercase_string_pre
  • x_su_lui ON lui
  • x_su_nsp ON norm_string_pre
  • x_su_sp ON string_pre
    • Insertion Role:When classes is loaded by the source insertion process, any strings that have never yet existed in the system are assigned SUI, LUI, and ISUI values and loaded into this table. This operation is performed by MEME_SOURCE_PROCESSING.assign_string_uis.
