mrd_attributes

Overview/Objective

This document contains a description of mrd_attributes and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:mrd_attributes
Table Type: unknown
Description:This table tracks mrd attribute states. New states are derived from the attributes table.
Fields:
attribute_level VARCHAR2(1) must be in level_status_rank.level_value where table_name = 'A' 
ui VARCHAR2(10)  
cui VARCHAR2(10) a Concept Unique Identifier, e.g. C0000039 
lui VARCHAR2(10) must be in string_ui.lui 
sui VARCHAR2(10) must be in string_ui.sui 
sg_type VARCHAR2(50) a field indicating the type of an sg_id, valid values can be found in code_map.code where the type='sg_type' 
suppressible VARCHAR2(10) must be in suppressible_rank.suppressible 
attribute_name VARCHAR2(50) the name of this attribute, e.g. SEMANTIC_TYPE 
attribute_value VARCHAR2(100) the value of this attribute if it can be represented in less than 100 characters. Otherwise it is a pointer into the stringtab.string_id field, e.g. <>Long_Attribute<>:12345 
string_id NUMBER(12) an id that groups fragments of a long string together 
code VARCHAR2(50) a value assigned to this atom/term by the source provider, this value has semantics specific to that source provider 
root_source VARCHAR2(20) a value representing a source independent of version which can be used across time to mean the same source 
atui VARCHAR2(12) must be in attributes_ui.aui 
source_atui VARCHAR2(50) a source asserted unique id for each attribute as it appears in the native source files 
hashcode VARCHAR2(100)  
insertion_date DATE indicates the date this row was inserted 
expiration_date DATE  
Indexes:
  • x_ma_atn ON attribute_name
  • x_ma_cui ON cui
  • x_ma_d ON insertion_date, expiration_date
  • x_ma_id ON atui
