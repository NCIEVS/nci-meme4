mom_precomputed_facts

Overview/Objective

This document contains a description of mom_precomputed_facts and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:mom_precomputed_facts
Table Type: source
Description:The repository of facts loaded from mergefacts files. The $MEME_HOME/bin/load_src.csh script can be used to load a mergefacts.src file into this table. From here it is processed and loaded into mom_candidate_facts.
Fields:
status VARCHAR2(1) Used to determine if this fact has completely mapped atom_id and concept_id values 
atom_id_1 NUMBER(12) must be in classes.atom_id or in dead_classes.atom_id 
atom_id_2 NUMBER(12) must be in classes.atom_id or in dead_classes.atom_id 
sg_id_1 VARCHAR2(50) a generic type if identifier, originally for source group identifier 
sg_type_1 VARCHAR2(50) a field indicating the type of an sg_id, valid values can be found in code_map.code where the type='sg_type' 
sg_qualifier_1 VARCHAR2(50) some sg_types require two pieces of information, an id and a qualifier, e.g. for CODE_SOURCE it would be a source value 
sg_id_2 VARCHAR2(50) a generic type if identifier, originally for source group identifier 
sg_type_2 VARCHAR2(50) a field indicating the type of an sg_id, valid values can be found in code_map.code where the type='sg_type' 
sg_qualifier_2 VARCHAR2(50) some sg_types require two pieces of information, an id and a qualifier, e.g. for CODE_SOURCE it would be a source value 
merge_level VARCHAR2(3) a value indicating the type of merge fact, usually one of MAT (exact match), NRM (norm match), or SY (synonym) 
source VARCHAR2(20) must be in source_rank.source 
integrity_vector VARCHAR2(1500) a vector of integrity checks, e.g. <MGV_C:V><MGV_H1:V> 
make_demotion VARCHAR2(1) a Y/N flag indicating whether or not demotion relationships should be created if this merge fact fails 
change_status VARCHAR2(1) a Y/N flag indicating whether or not the atoms merged by this fact should be unapproved 
merge_set VARCHAR2(30) the name of the merge set that this fact belongs to 
Indexes: No indexes
Insertion Role:This table is loaded from mergefacts.src by the script /bin/load_src.csh. Once the table is loaded, the sg_id fields are mapped to valid atom_id and concept_id fields by MEME_SOURCE_PROCESSING.prepare_src_mergefacts.
