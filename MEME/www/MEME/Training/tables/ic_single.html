ic_single

Overview/Objective

This document contains a description of ic_single and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:ic_single
Table Type: integrity
Description:Tracks data for unary integrity checks. A check like MGV_B requires a single piece of additional information which would be stored in this table.
Fields:
ic_name VARCHAR2(10) name of an integrity check, e.g. DT_M1 
negation VARCHAR2(1) a Y/N flag indicating whether to perform a boolean not operation on this check 
type VARCHAR2(20) SOURCE or TERMGROUP 
value VARCHAR2(100) a value of type type 
Indexes: No indexes
Insertion Role:A major insertion process is the running of the merge engine which often takes a set of merge inhibitors which it uses to prevent certain merges. Some merge inhibitors are known as unary integrity checks which means that they require one pieces of information drawn from a list in order to function (e.g. MGV_B which requires a list of sources). That information comes from this table.
