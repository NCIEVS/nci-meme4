events_processed

Overview/Objective

This document contains a description of events_processed and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:events_processed
Table Type: unknown
Description:Once the MRDStateManager processes events from the event_queue, they are moved into this table for permanent storage. They are removed at that point from event_queue.
Fields:
mrd_event_id NUMBER(12)  
mid_event_id NUMBER(12) This is either a molecule_id (for Undo/Redo actions) or an id generated in the same id space (from the MOLECULAR_ACTIONS row of max_tab) used to ensure that data changes to core and non-core tables are coordinated 
authority VARCHAR2(20) the authority responsible for this row, can be validated by MEME_RANKS.get_source_authority_rank 
timestamp DATE indicates when this row was inserted or last modified 
dtd_name VARCHAR2(20)  
dtd_version VARCHAR2(20)  
document CLOB an XML document or document fragment 
Indexes:
  • ep_pk ON mrd_event_id
  • x_ep_mid ON mid_event_id
