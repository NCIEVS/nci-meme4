action_log

Overview/Objective

This document contains a description of action_log and its various fields. It also (may) include information about how this table is used in the various stages of MID processing.
 

Details

Table name:action_log
Table Type: action
Description:Tracks all non-implied actions in the database. Every data change negotiated by MEME4 will be logged here.
Fields:
action_id NUMBER(12)  
transaction_id NUMBER(12) must be in molecular_actions.transaction_id 
work_id NUMBER(12) must be either 0 or must be in meme_work.work_id 
undo_action_id NUMBER(12)  
elapsed_time NUMBER(12) an integer number of milliseconds 
action VARCHAR2(500) a code indicating the action performed, must be in code_map where the type='atomic_action' 
synchronize VARCHAR2(1)  
authority VARCHAR2(50) the authority responsible for this action 
timestamp DATE indicates when this row was inserted or last modified 
document CLOB an XML document or document fragment 
Indexes:
  • ael_pk ON action_id
    • Insertion Role:Each insertion operation that changes the database will create an entry.
