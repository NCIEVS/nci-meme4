MEME4 Properties File

Following is documentation on the properties contained in the Java properties file used by MEME.
 
General Information The default properties file used by the MEME can typically be found in $MEME_HOME/bin/meme.prop.  Most MEME applications will support an alternative properties file via the -prop=<file> switch.  This file is so important MEME application server operation that the default properties file is a releasable component 
   
Server Properties
meme.server.port
Specifies the port on which the server should run.
meme.server.bootstrap.classes
A list of Initializable classes that are used to bootstrap the server.
meme.server.services.classes
A list of MEMEApplicationService classes that provide server functionality once it is started.  If this list changes, the server must be shut down and restarted.
meme.server.session.timeout
If a timeout is not specified by a request, the default time (in milliseconds) of inactivity before a session is programatically timed out.
meme.threadpool.size.optimal
Optimal size of the ThreadPool used by the server.
meme.threadpool.size.max
Maximum size of the ThreadPool used by the server.
meme.theadpool.autoextend
Indicates whether or not the thread pool should be allowed to auto-extend.
meme.datasourcepool.size.optimal
Optimal size of the MIDDataSourcePool used by the server.
meme.datasourcepool.size.max
Maximum size of the MIDDataSourcePool used by the server.
meme.datasourcepool.autoextend
Indicates whether or not the data source pool should be allowed to auto-extend.
meme.validate.actions.atomic
Flag indicating whether or not the server is validating atomic actions.
meme.validate.actions.molecular
Flag indicating whether or not the server is validating molecular actions.
Client Properties
meme.client.server.host, meme.client.server.port
Used by the client indicate the host and port running the application server. The default values if these are not specified come from the meme-server-host and meme-server-port mid services keys.
meme.client.protocol.class
The implementation of MEMERequestClient used to talk to the server.  The class used here must match the MEMERequestListener used by the server.
Mail Properties
meme.admin.midsvcs
Email address of the mid services administrator.
meme.admin.database
Email address of the database administrator.
meme.admin.system
Email address of the system administrator.
meme.admin.meme
Email address of the MEME application server administrator.
meme.smtp.host
The mail server used by MEMEMail.
Data Source Properties
meme.mid.connection.class
Specifies the class that the MEMEDataSourceFactory should use to create a connection to a MIDDataSource.
meme.mid.driver.class
The JDBC driver class used to connect to the MID database.
meme.mid.service.default
The mid services name used to represent the MID database that will be used. Typically this is expressed as a mid services name sans the -tns or -jdbc suffix (e.g. current-editing).
meme.mid.user.default
The username with which to connect to the MID database.
meme.mid.password.default
The password accompanying the username with which to connect to the MID database.
meme.server.schedule.service
A flag indicating which data source the MEME schedule should be connected to (if any). The two valid values are MID and MRD.

 
LVG/MID Services Properties
meme.mid.service.host, meme.mid.service.port
The host and port of the mid services socket server.
meme.lvg.host
The host of the LVG socket server.  This server supports norm, luiNorm, and wordind functions.
General Properties
meme.view
A flag indicating whether or not the application should open windows or run on the command line. Valid values are true and false.
meme.debug
A flag indicating whether or not the application should run in debug mode (meaning that it will print out lots of extra trace information). Valid values are true and false.
meme.log.path
The name of the file to which logging information should be written. If left blank or unset, STDOUT is used instead.
meme.env.MEME_HOME, meme.env.ORACLE_HOME
These are used to represent the environment variables $MEME_HOME and $ORACLE_HOME. This is typically set by the perl wrapper scripts so it is probably not necessary to set it in the log file.
meme.tmp.directory
The name of a directory where temporary files can be created. Typically this is just /tmp.
meme.dtd.directory
The name of a directory containing document type definitions.
meme.properties.file
The path to the properties file to use.
