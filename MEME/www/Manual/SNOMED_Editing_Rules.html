SNOMED Editing Rules

10/31/2003 - SNOMED Codes

We merged SNOMED CT based on legacy codes from the old SNMI98 and RCD99. We were told that the meanings did not change from the legacy code concepts to the new SNOMED CT. We have discovered that this is not true in some cases. So, there will be times when merging was done on legacy codes but it was not correct. The old SNMI98 atoms do not mean the same as the new SNOMED CT atoms and should not be in the same concept. Same with RCD99. We are keeping the SNMI98 and RCD99 sources around and they are considered separate sources than SNOMED CT. We have found this to be particularly true with SNMI98 HT and HX terms and with measurements vs. lab results. 

An example is (made up): 
 Incision of wall of atrium (SNOMED CT)
General operative procedures of heart (SNMI98/HT/D-34670)
 
These are not the same even though the new SNOMED CT is linked to the legacy code of the SNMI98 terms. They need to be split.

Now for synonymy and splitting.....if SNOMED CT has a bunch of atoms with the same code and the concept only has SNOMED CT atoms but you do not agree with the synonymy, leave alone unless the synonymy is really, really really wrong.

10/29/2003 - Parentheses

Just a reminder, all the SNOMED CT FN atoms have words in parentheses that help explain what the meaning of the atom is....like procedure, device, finding, etc.

For clinical editing, there really should not be many if any cases were there are two SNOMED FN atoms in a concept and they have really divergent meanings in parentheses. What I mean is this:

disease X (finding)
Disease X (disorder)

This is fine because I sent you the merging rules and we are merging finding and disorders. Same with morph abnormality and disorder, etc.

But, if you see:

Blah X (procedure)
Blah X (device)

or

Blah X (disorder)
Blah X (device)

This is a big red flag that these two different SNOMED concepts should not be in the same concept.

10/27/2003 - Organisms

Only for SNOMED CT organisms....

If you have a concept with 2 or more SNOMED CT codes in it, and they are merged because NCBI says that two or more organism names are the same (based on them having the same NCBI code), go ahead and respect NCBI for now. We will go over these again when we are done editing.

If 2 or more SNOMED CT codes are in the same concept, and there is no NCBI indicating synonymy, these need to be looked at.

We are trying to respect SNOMED CT as much as possible. So, if you want to merge or split some SNOMED CT codes and it is not because of NCBI, that needs to be checked out. Some simple google searches can usually confirm or deny synonymy.

Original SNOMED editing rules:

The term types for SNOMED CT are:

SNOMEDCT_2003_07_31/FN      Fully-specified name
SNOMEDCT_2003_07_31/IS      Obsolete synonym
SNOMEDCT_2003_07_31/OF      Obsolete fully-specified name
SNOMEDCT_2003_07_31/OP      Obsolete preferred term
SNOMEDCT_2003_07_31/PT      Preferred term
SNOMEDCT_2003_07_31/PTGB        Preferred term (Great Britain)
SNOMEDCT_2003_07_31/SB      Subset name
SNOMEDCT_2003_07_31/SY      Synonym
SNOMEDCT_2003_07_31/SYGB        Synonym (Great Britain)
SNOMEDCT_2003_07_31/XM      Crossmap set

The FN term type has the highest precedence. Many will have something in brackets in the name...example: Gum Diseases (disorder) or Excision (Procedure). The description in
brackets helped with the merging below. The PT though is the preferred name and appears in the context in the reports.

In the concept reports, you will see legacy codes from SNMI98 and RCD. These legacy codes were used to help in merging. The legacy codes should have the same meaning as the
concept that refers to it. IF there are several concepts merged together, you may need to check the Clue Browser to figure out what is going on. In rare cases that they don't have
the same meaning, we need to report these to SNOMED. Send Tammy and Laura an e-mail if  you think the new codes did not retain the same meaning as the legacy codes.

Rules for merging duplicate strings based on categories: These were the rules Apelon used when inserting SNOMED. In some cases, you may decide that the merges are incorrect and
need to be split. In almost all cases you want to keep everything with the same code together from SNOMED CT. If you decide you need to split, send Tammy and Laura an
email first.

"Do Not Merge" means do not merge 2 Snomed PTs (and the SYs and fully specified terms associated with them) into the same concept
during the insertion. It does not mean, do not merge into existing concepts w/in the mid.

1. body structure/body structure
do not merge, do not create a demotion

For the concepts from SNOMED with the PT containin word "Entire" i.e. Entire Liver, only merge if the PT contains an exact or norm match to something already in the mid.
Do not use the SYs for merging.
If a multi-meaning occurs, a PN matching the PT should be created. In the case of liver, Snomed has a rela between the two concepts (is_a).

The concept with the word "structure" can merge into existing concepts base on on PT, SY, or legacy code. If a PN doesn't already exist, we think the PN should be the same as the SY. Leftover from RCD but still in SNOMED:

[V] terms AND all ICD9 V codes AND all ICD10 codes under 'Factors influencing health status  and contact with health services' MAY have DIFFERENT meanings from the apparent meaning and should be Multiple Meanings in those cases.

The classic example is:

'V01.0 Cholera'
which means
'V01  Contact with or exposure to communicable diseases'
     

Read [V] terms; ICD-9 V codes;
ICD10 'Factors influencing health status and contact with health services'
------------------------------------------------------------
[V] and all ICD V codes are 'Health Status and contact with health
services factors' in Read;  the ICD-9-CM manual says:

  'This classification is provided to deal with occasions when
   circumstances other than a disease or injury classifiable
   to categories 001-999 (the main part of ICD) are recorded as
   "diagnoses or "problems."

and goes on to list cases where a person not currently sick
needs:

  a) prohylactic vaccination, to discuss a problem which is not a
     disease or injury, etc.;
  b) when a person is receiving treatment (e.g. dialysis for
     renal disease, chemotherapy for malignancy, cast change
  c) circumstance which  which influences health status
     but is not a disease or injury.

    * Meaning of Read bracketed codes
